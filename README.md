# Folio

Folio is a local-first markdown viewer and light editor for macOS. It renders markdown files with the careful typography and clean design of a printed document, giving reading primacy over editing. Files live in a folder you control — on your computer, in Google Drive, or elsewhere — with no database, no lock-in.

## Features

- **Full-featured rendering:** GFM (tables, task lists, strikethrough, autolinks, footnotes), fenced code with syntax highlighting, LaTeX maths, Mermaid diagrams, GitHub-style callouts (note/warning/tip/important/caution), images
- **Reader-first design:** Opens in reading mode; ⌘E toggles a lightweight source editor; search and navigate with ⌘K (quick open) and ⇧⌘F (full-text search)
- **Links as navigation:** Click relative markdown links (`[file](./other.md)`), Obsidian-style wikilinks (`[[title]]`), or anchors to jump between files; ⌘[ and ⌘] navigate browser-style history
- **Live reload:** Edit a file in another app — or sync via Google Drive — and watch the viewer refresh within a second
- **Interactive checkboxes:** Tick task lists in the reader; the change persists to disk immediately
- **Conflict-safe edits:** Atomic writes with mtime checks mean simultaneous edits (e.g. Drive sync vs. your changes) surface a non-modal banner; never silent data loss
- **Text zoom:** ⌘+ / ⌘− to scale the document; current size displayed in the footer
- **Print to PDF:** ⌘P produces a beautiful PDF straight from the view, with typography intact

## Getting started

### Development

```bash
cd folio

# Frontend only (browser mode; serves sample vault from memory, no Tauri window)
npm install
npm run dev

# Full app in development mode (macOS only)
npm run tauri dev

# Type-check the TypeScript
npm run typecheck

# Build frontend for production
npx vite build

# Preview production build in Chromium
npx vite preview --port 4173
```

Browser mode (`npm run dev` + Chromium) includes the full link routing, conflict-banner simulation, and search functionality, so it is the primary test bed on Linux. Regression tests run against `vite preview` (see below).

### Building

```bash
# macOS only (Intel or Apple Silicon)
npm run tauri build

# This produces an unsigned .app and .dmg in src-tauri/target/release/bundle/
```

Release builds on Intel are slow (~10 minutes); prefer building on the M5 Air if possible.

### Project layout

- `src/App.tsx` — top-level three-pane shell (file tree, document, TOC); state and sync orchestration
- `src/markdown/` — unified pipeline: remark plugins (wikilinks, callouts), rehype transforms (Shiki, KaTeX, Mermaid lazy-load), post-mount DOM fixups
- `src/components/` — React components (Reader, Editor, Tree, SearchPanel, QuickSwitcher, ConflictBanner)
- `src/vault.ts` — storage facade; delegates to `ipc` in Tauri mode, serves samples from in-memory map in browser mode
- `src/linkRouter.ts` — resolves link clicks into app navigation
- `src-tauri/src/commands.rs` — Rust IPC: file I/O, watching, search, tree building, link index
- `src-tauri/icons/` — app icon files (generated by `npx tauri icon`)
- `samples/` — 10-file linked vault used as the rendering and link-routing fixture

## Testing

Regression tests run against a live `vite preview` server using Playwright in Chromium. In the `scratchpad` directory (populated from a prior session):

```bash
# In folio/ after building
npx vite build
npx vite preview --port 4173 &

# In the scratchpad directory
node smoke.mjs        # Rendering: counts render, Mermaid, callouts, etc
node linktest.mjs     # Link routing: navigate, anchor-scroll, history
node edittest.mjs      # Editor + conflict handling
node searchtest.mjs   # Search panel and quick switcher
```

All four tests must pass before commit.

## Architecture

**Tauri 2 + React 18 + TypeScript**

- **Frontend** (React, TypeScript, Vite): Rendering pipeline, UI, event handling. Runs in WKWebView on macOS (not Chromium).
- **Backend** (Rust): File I/O, watching, search. All disk writes go through a single atomic write path with mtime conflict checking to handle concurrent edits (e.g. Google Drive sync).

The frontend and backend communicate via Tauri's IPC layer (`src/ipc.ts`). Browser mode (for testing on Linux) bypasses Tauri entirely and serves files from an in-memory vault built from the `samples/` directory, mirroring the real IPC semantics exactly.

## Design

Typographic, print-first, mostly monochrome: crisp white/light-grey paper (#FFFFFF), ink text (#16171A), hairline rules, plus a single restrained terracotta accent (#B0451F) for links, selection and active states. The file browser is a Finder-style column view. Fonts are bundled locally (Jost for headings, Lora for body, JetBrains Mono for chrome and code) so the app works offline.
